# select 比较早

  • synchronous I/O multiplexing 同步的io多路复用器
  • 1024的限制 man select 查看
  • 这个遍历的过程出发了一次系统调用，用户态内核态的切换，过程中把fds传给内核， 内核重新根据用户这次调用传过来的fds遍历修改状态
  • 循环 单调用
# 弊端:
  • 每次都要重新传递fds
  • 每次内核被调用以后，针对这次调用出发一个遍历fds全量的复杂度

# 中断（概念）

去中间向量表找（int 0-255，callback） 80就是中断 callback 回调函数

  • 硬中断（时钟中断，晶振）
  • 软中断（陷阱） int 80
  • io中断（网卡 鼠标 键盘等）

# epoll 之前的callback

  • 只是完成了网卡发来的数据走内核协议栈（2链路层，3网络层，4传输控制层）， 最终关联到fd的buffer里，所以你某一时间如果从app询问内核某一个或者 某些FD可能是有可R/W的，会有状态返回

  • 如果内核在callback处理中再加入链表？

# epoll create
  • 在内核开辟一个空间（红黑树）fd6
# epoll_ctl
  • 在fd6里面存一个fd4（read状态）(链表)
# epoll_wait
  • 遍历fd4链表(就不用全遍历了)

# 整理下

  • 数据先到达内存
  • 中断 callback
  • 将数据和fd关联起来(网卡的数据扔到buffer)
  • 中断延伸 处理了buffer以后延伸处理
  • 内核中程序处理完fd的buffer,状态 之后,继续把有有状态的fd copy到链表中
  • epoll_wait拿到 fds(链表) 然后accept
  • 当有N个fd有R/W的时候 (将N个FD分组 每一个组一个selector，将一个selector压到一个线程上)
